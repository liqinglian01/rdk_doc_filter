# 多维度内容复用 + 动态过滤显示（版本 × 产品）实现方案

## 1. 实现方案概述

本方案通过 React 组件和 Context API 实现了基于版本和产品的多维度内容复用和动态过滤显示，避免了内容重复，提高了文档维护效率。

### 1.1 核心组件

- **DocScopeFilterContext**：管理版本和产品状态，提供状态更新方法
- **DocScope**：根据版本和产品条件显示内容的组件
- **DocScopeSelectors**：版本和产品选择器组件
- **DocSidebar**：自定义侧边栏组件，根据版本和产品过滤目录项

### 1.2 技术架构

- 使用 React Context API 进行状态管理
- 使用 URL 参数存储和同步版本和产品选择
- 使用 MDX 组件实现条件显示

## 2. 核心文件结构

```
src/
├── components/
│   ├── DocScope.jsx          # 条件显示组件
│   └── DocScopeSelectors.jsx # 版本和产品选择器
├── context/
│   └── DocScopeFilterContext.js # 状态管理上下文
└── theme/
    ├── DocRoot/Layout/
    │   └── index.js          # 应用根布局
    └── DocSidebar/
        └── index.js          # 侧边栏过滤组件
```

## 3. 详细实现

### 3.1 版本和产品矩阵定义

在 `DocScopeFilterContext.js` 中定义了版本和产品的矩阵关系：

```javascript
/** 版本 → 该产品线下拉可选的产品名（与正文 DocScope 中 products 字符串一致） */
export const VERSION_PRODUCT_MATRIX = {
  '3.0.0': ['RDK X3', 'RDK X3 Module'],
  '3.5.0': ['RDK X5', 'RDK X5 Module'],
  '4.0.5': ['RDK S100'],
  '5.0.1': ['RDK S600'],
};
```

### 3.2 状态管理

`DocScopeFilterContext.js` 提供了：
- 版本和产品状态管理
- 从 URL 参数解析状态
- 状态更新方法

### 3.3 条件显示组件

`DocScope.jsx` 组件：
- 接收 `versions` 和 `products` 属性
- 根据当前选择的版本和产品决定是否显示内容
- 支持逗号分隔的多个版本或产品

### 3.4 选择器组件

`DocScopeSelectors.jsx` 组件：
- 显示版本和产品下拉选择器
- 与 URL 参数同步
- 版本切换时自动更新产品列表

### 3.5 应用集成

在 `DocRoot/Layout/index.js` 中：
- 使用 `DocScopeFilterProvider` 包裹整个应用
- 在文档顶部显示 `DocScopeSelectors` 组件

## 4. 使用方法

### 4.1 导入组件

在需要使用条件显示的文档中，首先导入 `DocScope` 组件：

```markdown
```mdx-code-block
import DocScope from '@site/src/components/DocScope';
```
```

### 4.2 基础用法

使用 `DocScope` 组件包裹需要条件显示的内容：

```markdown
<DocScope versions="3.0.0" products="RDK X3">
  这里是只在版本 3.0.0 中显示的 RDK X3 内容。
</DocScope>
```

### 4.3 多版本和多产品

使用逗号分隔多个版本或产品：

```markdown
<DocScope versions="3.0.0,3.5.0" products="RDK X3,RDK X5">
  这里是在版本 3.0.0 或 3.5.0 中显示的 RDK X3 或 RDK X5 内容。
</DocScope>
```

### 4.4 包裹警示卡片

可以包裹 Docusaurus 的警示卡片：

```markdown
<DocScope versions="3.0.0" products="RDK X3">

:::info
这里是只在版本 3.0.0 中显示的 RDK X3 内容。
:::

</DocScope>
```

### 4.5 不限制维度

如果不限制某个维度，可以不设置对应的属性或留空：

```markdown
<DocScope versions="3.0.0">
  这里是只在版本 3.0.0 中显示的内容，不限制产品。
</DocScope>

<DocScope products="RDK X3">
  这里是只在产品为 RDK X3 时显示的内容，不限制版本。
</DocScope>
```

## 5. 控制整个章节的显示

除了使用 `DocScope` 组件控制文档内容的显示外，还可以控制整个章节在侧边栏中的显示。这样可以避免在非目标版本/产品下出现空章节的问题。

### 5.1 实现原理

通过自定义 `DocSidebar` 主题组件，在侧边栏渲染时根据版本和产品条件动态过滤目录项。核心逻辑位于 `src/theme/DocSidebar/index.js`。

**工作流程：**

1. 侧边栏保持自动生成模式（`sidebars.js` 中使用 `type: 'autogenerated'`）
2. 自定义 `DocSidebar` 组件在渲染前拦截侧边栏数据
3. 根据 `SIDEBAR_SCOPE_MAP` 配置，匹配需要手动控制的章节
4. 匹配到的章节按配置的版本/产品规则决定是否显示
5. 未匹配到的章节默认自动生成并展示

### 5.2 核心配置

在 `src/theme/DocSidebar/index.js` 中定义 `SIDEBAR_SCOPE_MAP`：

```javascript
const SIDEBAR_SCOPE_MAP = {
  'rdk_x5': { versions: ['3.5.0'], products: ['RDK X5'] },
};
```

**配置说明：**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| **key** | `string` | 用于匹配侧边栏项的字符串，会匹配 `id`、`label`、`href`、`docId`（大小写不敏感） |
| **versions** | `string[]` | 该章节在哪些版本下显示，空数组 `[]` 表示所有版本都显示 |
| **products** | `string[]` | 该章节在哪些产品下显示，空数组 `[]` 表示所有产品都显示 |

### 5.3 使用方法

#### 步骤一：确认章节标识

首先确认需要控制的章节在侧边栏中的标识。通常与文档文件名相关，例如 `rdk_x5.md` 对应的侧边栏项 `id`、`label`、`href`、`docId` 中会包含 `rdk_x5`。

#### 步骤二：添加配置

在 `SIDEBAR_SCOPE_MAP` 中添加章节的显示规则：

```javascript
const SIDEBAR_SCOPE_MAP = {
  // rdk_x5 章节只在 3.5.0 版本 + RDK X5 产品下显示
  'rdk_x5': { versions: ['3.5.0'], products: ['RDK X5'] },

  // rdk_x3 章节只在 3.0.0 版本下显示，不限制产品
  'rdk_x3': { versions: ['3.0.0'], products: [] },

  // rdk_s100 章节只在 RDK S100 产品下显示，不限制版本
  'rdk_s100': { versions: [], products: ['RDK S100'] },

  // 某文档在多个版本下显示
  'shared_doc': { versions: ['3.0.0', '3.5.0'], products: ['RDK X3', 'RDK X5'] },
};
```

#### 步骤三（可选）：配合 DocScope 控制内容

如果还需要防止用户通过 URL 直接访问时看到不该看的内容，可以在对应的 Markdown 文件中使用 `DocScope` 组件包裹内容：

```markdown
---
sidebar_position: 2
---

```mdx-code-block
import DocScope from '@site/src/components/DocScope';
```

<DocScope versions="3.5.0" products="RDK X5">

# 1.1.2 RDK X5

这里是 RDK X5 的文档内容...

</DocScope>
```

> **是否需要 DocScope？** 如果整个文件都是某个版本/产品的专属内容，且侧边栏已隐藏入口，用户不会主动访问到，则 `DocScope` 不是必须的。只有在需要防止 URL 直接访问时才需要配合使用。

### 5.4 匹配规则

`SIDEBAR_SCOPE_MAP` 的 key 会与侧边栏项的以下字段进行**包含匹配**（大小写不敏感）：

- `item.id`：文档 ID，通常由文件路径生成
- `item.label`：侧边栏显示的标签文本
- `item.href`：文档链接路径
- `item.docId`：文档唯一标识

只要任一字段包含 key 字符串，即视为匹配成功。

**匹配示例：**

| key | item.id | item.label | 是否匹配 |
| --- | --- | --- | --- |
| `rdk_x5` | `01_Quick_start/hardware_introduction/rdk_x5` | `1.1.2 RDK X5` | ✅ 匹配（id 包含 rdk_x5） |
| `rdk_x5` | `tutorial/rdk_x5_module` | `RDK X5 Module` | ✅ 匹配（id 包含 rdk_x5） |
| `rdk_x5` | `tutorial/rdk_x3` | `RDK X3` | ❌ 不匹配 |

### 5.5 分类（Category）的处理

如果被控制的文档位于某个分类目录下，过滤逻辑会递归处理：

- 如果分类下的所有子项都被过滤掉，该分类也会被隐藏
- 如果分类下仍有子项显示，该分类保留
- 分类本身不会根据 `SIDEBAR_SCOPE_MAP` 进行匹配，只有叶子节点（文档项）才会

### 5.6 与 DocScope 的配合关系

| 层级 | 控制方式 | 作用 |
| --- | --- | --- |
| **侧边栏** | `SIDEBAR_SCOPE_MAP` | 控制章节是否在目录中显示 |
| **内容** | `DocScope` 组件（可选） | 控制文档内容是否渲染 |

**配合说明：**
- 只配置 `SIDEBAR_SCOPE_MAP`：侧边栏中章节隐藏，用户无法通过导航访问。但通过 URL 直接访问仍能看到内容
- 只使用 `DocScope` 但不配置 `SIDEBAR_SCOPE_MAP`：内容隐藏了，但侧边栏中仍显示空章节
- **两者同时配置**：侧边栏和内容都正确过滤，即使通过 URL 直接访问也看不到不该看的内容

> **实践建议**：如果整个文件都是某个版本/产品的专属内容（如 `rdk_x5.md`），通常只配置 `SIDEBAR_SCOPE_MAP` 就够了，因为侧边栏隐藏后用户不会主动访问到。如果文件中混合了多个版本/产品的内容，则需要 `DocScope` 来控制内容级别的显示。

## 6. 优势

1. **避免内容重复**：不同版本和产品的内容可以在同一个文件中管理
2. **动态过滤显示**：根据用户选择的版本和产品显示相应的内容
3. **URL 同步**：用户可以通过 URL 直接访问特定版本和产品的内容
4. **易于维护**：不需要为每个版本和产品创建单独的文件
5. **灵活配置**：可以根据需要轻松添加新的版本和产品
6. **自动+手动结合**：侧边栏自动生成，同时支持手动控制特定章节的显示

## 7. 注意事项

### 7.1 DocScope 组件相关

1. 版本和产品的字符串必须与 `VERSION_PRODUCT_MATRIX` 中定义的一致
2. 多个版本或产品之间用逗号分隔
3. 空字符串或不设置 `versions` 和 `products` 属性表示不限制该维度
4. 确保在文档中正确导入 `DocScope` 组件

### 7.2 章节显示控制相关

1. **`SIDEBAR_SCOPE_MAP` 的 key 选择要精确**：key 用于包含匹配，如果 key 太短或太通用（如 `rdk`），可能会误匹配其他章节。建议使用文件名级别的标识，如 `rdk_x5`、`rdk_x3`
2. **匹配是大小写不敏感的**：`rdk_x5` 可以匹配 `RDK X5`、`rdk_x5`、`Rdk_X5` 等
3. **注意子串匹配的副作用**：`rdk_x5` 会同时匹配 `rdk_x5` 和 `rdk_x5_module`。如果需要区分，请使用更精确的 key 或调整文件命名
4. **`SIDEBAR_SCOPE_MAP` 和 `DocScope` 建议配合使用**：只配置 `SIDEBAR_SCOPE_MAP` 时侧边栏隐藏但 URL 仍可访问内容；只使用 `DocScope` 时内容隐藏但侧边栏显示空章节。对于整个文件专属某版本/产品的场景，只配置 `SIDEBAR_SCOPE_MAP` 通常即可
5. **版本和产品字符串必须与 `VERSION_PRODUCT_MATRIX` 中定义的完全一致**：包括空格和大小写
6. **空数组 `[]` 表示不限制**：`versions: []` 表示所有版本都显示，`products: []` 表示所有产品都显示
7. **分类目录会自动处理**：如果分类下所有子项都被过滤掉，该分类也会自动隐藏，无需额外配置
8. **未在 `SIDEBAR_SCOPE_MAP` 中配置的章节默认显示**：只有显式配置的章节才会被控制，其他章节保持自动生成和展示
9. **修改 `SIDEBAR_SCOPE_MAP` 后需要刷新页面**：由于 Docusaurus 的热更新机制，修改配置后可能需要刷新浏览器才能看到效果

## 9. 扩展和维护

### 9.1 添加新的版本和产品

要添加新的版本和产品，只需修改 `DocScopeFilterContext.js` 中的 `VERSION_PRODUCT_MATRIX`：

```javascript
export const VERSION_PRODUCT_MATRIX = {
  '3.0.0': ['RDK X3', 'RDK X3 Module'],
  '3.5.0': ['RDK X5', 'RDK X5 Module'],
  '4.0.5': ['RDK S100'],
  '5.0.1': ['RDK S600'],
  '5.1.0': ['RDK S700'], // 新增版本和产品
};
```

### 9.2 添加新的章节显示控制

要控制新章节的显示，只需在 `src/theme/DocSidebar/index.js` 的 `SIDEBAR_SCOPE_MAP` 中添加配置：

```javascript
const SIDEBAR_SCOPE_MAP = {
  'rdk_x5': { versions: ['3.5.0'], products: ['RDK X5'] },
  // 新增：rdk_s100 章节只在 4.0.5 版本 + RDK S100 产品下显示
  'rdk_s100': { versions: ['4.0.5'], products: ['RDK S100'] },
};
```

同时在对应的 Markdown 文件中使用 `DocScope` 组件包裹内容。

### 9.3 自定义样式

可以通过 CSS 自定义 `DocScope` 组件和 `DocScopeSelectors` 组件的样式：

```

## 10. 总结

本方案通过 React 组件和 Context API 实现了基于版本和产品的多维度内容复用和动态过滤显示，避免了内容重复，提高了文档维护效率。使用方法简单直观，支持多版本和多产品的组合，并且与 URL 同步，提供了良好的用户体验。

方案包含两个层面的过滤控制：
- **内容层面**：通过 `DocScope` 组件控制文档内容的条件显示
- **侧边栏层面**：通过自定义 `DocSidebar` 组件和 `SIDEBAR_SCOPE_MAP` 配置控制章节在目录中的显示

两者配合使用，确保用户在不同版本和产品下只看到相关的章节和内容，避免出现空章节或不一致的情况。

通过这种方式，文档维护者可以在同一个文件中管理不同版本和产品的内容，减少了重复工作，同时用户可以根据自己的需求快速切换到相应的版本和产品文档。
